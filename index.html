PAC 在线生成器

自动分流 · 国内/私有 IP 直连 · 国外走代理 · 多代理故障转移

① 代理服务器 故障转移链

按顺序填写你的代理。生成的 PAC 会让系统自动依次尝试，不可用时自动跳到下一个（即“自动轮询可用代理”）。

全部代理不可用时直连
末尾追加 DIRECT 兜底，避免断网
按当前网络自动优选代理
同网段代理（如家里/公司 LAN）自动置顶，避免先连死代理再转移

② 路由规则

基于 gfw-pac 改造：中国 IP 段（5823 条）走直连，私有/本地地址直连，其余国外流量走代理。

中国 IP 直连
命中国内 IP 段 → DIRECT
私有 IP 直连
10/172.16-31/192.168/127/169.254 等
本地 / localhost 直连
纯主机名与 .test/.localhost
国外 IP 走代理
未命中上述规则 → 代理链
调试弹窗
每次请求弹窗显示命中规则（仅排查用）

③ 自定义域名

已预填常用直连/代理域名，可按需增删。每行一个，支持子域通配（如 baidu.com 同时匹配 www.baidu.com）。

④ 中国 IP 列表

内置 5823 条中国 IPv4/IPv6 段（来源 Loyalsoldier/geoip）。可选：填入远程列表 URL 即时更新。

⑤ 生成的 PAC 实时

编辑上方任意项即时刷新。下载 pac.js 后指向系统/浏览器代理即可。


      

        
        
        
      

      

        如何使用生成的 PAC？
        
    
          
  1. 点击「下载 pac.js」，把文件保存到本地（如 ~/pac.js）。
    2. 
          
  2. 系统级：macOS 系统设置 → 网络 → 代理 → 自动代理配置 → 填入 file:///Users/你/pac.js；Windows 设置 → 代理 → 使用设置脚本 → 同上。
    3. 
          
  3. 浏览器：Chrome/Edge 扩展（如 SwitchyOmega）选择「PAC 模式」并指向该文件；或浏览器设置中填写 PAC 地址。
    4. 
          
  4. 想用在线地址：把生成的 pac.js 提交到本仓库，访问 https://<用户名>.github.io/<仓库名>/pac.js 即可作为 PAC URL。
    5. 
          
  5. 调试：打开「调试弹窗」开关重新下载，浏览器会为每个请求弹出命中规则，方便排查。
    6.
