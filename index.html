📖 协议说明 & 常用工具对照（HTTP / HTTPS / SOCKS4 / SOCKS5 怎么选？）
HTTPPROXY

最基础、兼容性最好。浏览器把请求交给代理代为访问，仅 TCP。绝大多数本地软件（Clash / V2Ray）都开放 HTTP 端口，PAC 里最常用。

HTTPSHTTPS

“CONNECT 隧道代理”。客户端先与代理建立 TLS 加密通道再传请求，代理看不到明文。本机 127.x 代理一般没必要用，多用于远端代理服务器。

SOCKS4SOCKS

古老协议：仅 IPv4、无认证、不支持 UDP，基本已被淘汰。除非老软件只支持它，否则不要用。

SOCKS5SOCKS5

现代 SOCKS：支持 IPv4/IPv6、UDP（游戏 / 视频通话 / QUIC 需要）、可选认证。本地软件几乎都优先提供 SOCKS5，最通用。

        如何使用生成的 PAC？
        
    
          
  1. 点击「下载 pac.js」，把文件保存到本地（如 ~/pac.js）。
    2. 
          
  2. 系统级：macOS 系统设置 → 网络 → 代理 → 自动代理配置 → 填入 file:///Users/你/pac.js；Windows 设置 → 代理 → 使用设置脚本 → 同上。
    3. 
          
  3. 浏览器：Chrome/Edge 扩展（如 SwitchyOmega）选择「PAC 模式」并指向该文件；或浏览器设置中填写 PAC 地址。
    4. 
          
  4. 想用在线地址：把生成的 pac.js 提交到本仓库，访问 https://<用户名>.github.io/<仓库名>/pac.js 即可作为 PAC URL。
    5. 
          
  5. 调试：打开「调试弹窗」开关重新下载，浏览器会为每个请求弹出命中规则，方便排查。
